# Основной ИИ-контент-оркестратор

Дочерний поток, при этом главный n8n-сценарий, который выбирает подходящий инфоповод, очищает входные данные, оценивает качество кандидатов и передаёт готовый контекст в сценарии публикации для нескольких независимых проектов.

## Что делает сценарий

Сценарий берёт входящие публикации, с помощью code-узлов отсекает мусор, дубли и слабые сигналы, определяет тему, считает приоритет, выбирает лучших кандидатов и передаёт их в ИИ-редактор.

После генерации сценарий проверяет результат и запускает нужные дочерние сценарии для публикации в соцсетях разных проектов.

## Что реализовано

- очистка входящего текста от ссылок, упоминаний, тикеров и мусора с помощью code-узла;
- фильтрация слабых и бессмысленных публикаций с помощью code-узла;
- определение темы публикации с помощью code-узлов;
- скоринг кандидатов по вовлечённости, свежести и содержательности с помощью code-узла;
- группировка тем по приоритетам с помощью code-узла;
- выбор лучших кандидатов с помощью code-узла;
- ИИ-редактор с конкретным JSON-выводом;
- проверка результата модели;
- защита от ответов-отмазок ИИ вида “не могу открыть ссылку”;
- управление лимитами публикаций;
- Telegram-уведомления администратора об ошибках и статусах.
- cooldown-логика между проектами для исключения одновременной публикации для всех;
- запуск дочерних n8n-сценариев.

## Мой вклад

Разработал архитектуру и собрал поток от логики отбора и маршрутизации контента, до JavaScript-кода для фильтрации, скоринга, определения тем, антидублей и управления частотой публикаций.

Также настроил ИИ-узел, который превращает выбранный инфоповод в готовый контент для разных каналов, и добавил проверки, чтобы модель не возвращала бесполезные ответы вместо результата.
